# Scripts de DM — LinkedIn Outbound
*IRBIS · Frente B (founders de SaaS) · Criado em 08/jul/2026*

> Regras fixas antes de usar:
> - Auditar o site do prospect NO DIA do envio. Se corrigiram o problema, adapte o DM.
> - Conectar ANTES de enviar DM (sem conexão, a mensagem vai para "Solicitações").
> - Assinar sempre "Nicolas". Nunca "Equipe IRBIS".
> - Preço NUNCA em DM.
> - Follow-up nunca antes do D+4. Teto: 2 tentativas após o DM principal.

---

## SOLICITAÇÃO DE CONEXÃO (<300 chars)

**Com gatilho (rodada, lançamento, contratação):**
```
Vi que a [Empresa] [fechou rodada / lançou / cresceu X%]. Faço sites para founders de SaaS. Vou conectar.
```

**Sem gatilho específico:**
```
Acompanho o que a [Empresa] está construindo. Faço sites para founders. Vou conectar.
```

---

## DM PRINCIPAL (até 24h após aceitar)

> Regra: cite UM problema específico que você encontrou no site deles.
> Máx. 3 blocos de texto. Não explique o projeto. Não mencione preço.

**Variação A — Problema técnico/visual claro:**
```
[Nome], auditei o site de vocês antes de mandar isso.

[PROBLEMA ESPECÍFICO — ver exemplos abaixo]

Faço só sites. Entrego em semanas. Faz sentido conversar 20 minutos?

Nicolas
```

**Variação B — Founder pós-rodada ou marco recente:**
```
[Nome], parabéns pelo [marco].

Founders nesse estágio costumam ter produto que avançou mais rápido que o site. Vi isso no de vocês: [PROBLEMA ESPECÍFICO].

Faço só sites. Entrego em semanas. Faz sentido conversar 20 minutos?

Nicolas
```

**Variação C — Site genérico, sem problema técnico óbvio:**
```
[Nome], olhei o site de vocês antes de mandar isso.

O headline descreve a feature, não o que o cliente ganha. O visitante precisa ler o segundo parágrafo para entender o produto.

Faço só sites. Entrego em semanas. Faz sentido conversar 20 minutos?

Nicolas
```

### Banco de problemas específicos (use o que você encontrar no site deles)

| Problema encontrado | Como citar no DM |
|---|---|
| CTA some no mobile | "O CTA principal de vocês some no mobile. O formulário quebra antes do envio." |
| Headline descreve feature, não resultado | "O headline descreve a feature, não o que o cliente ganha. O visitante precisa ler o segundo parágrafo para entender o produto." |
| Prova social sem número | "A prova social de vocês não tem nenhum número. Logos sem contexto não convencem." |
| Preços aparecem antes do valor estar estabelecido | "A página de preços aparece no menu antes de qualquer argumento de valor. O visitante chega lá sem saber por que vale a pena." |
| Formulário genérico mistura lead com suporte | "O formulário de contato não diferencia lead de suporte. Quem quer contratar espera na mesma fila de quem tem dúvida técnica." |
| Site lento | "O site de vocês carrega em [X]s no mobile. 53% dos usuários abandonam em mais de 3 segundos." |

## FOLLOW-UP D+4 (se não respondeu)

> Entregue o lead magnet como valor. Não "bump". Não "só passando por".

```
[Nome], tudo bem se não for o momento.

Levantei 5 erros que afastam leads qualificados antes de chegarem no time de vendas. Um deles apareceu no site de vocês.

Quer o checklist?

Nicolas
```

```

```

Se disser sim, mande o case E-Force (+R$350k em vendas gerados).

## BREAKUP D+10

```
[Nome], encerro de minha parte.

Se site virar prioridade, me chame.

Nicolas
```

---

## FLUXO COMPLETO

```
D+0   Solicitação de conexão
      ↓ (aceita)
D+0/1 DM principal com problema específico
      ↓ (sem resposta)
D+4   Follow-up com checklist
      ↓ (sem resposta)
D+10  Breakup
```

Qualquer resposta do lead interrompe a cadência. Não mande o D+4 se ele respondeu, mesmo que a resposta seja negativa — trate separado.

---

## QUANDO RESPONDER COM INTERESSE

- Responda em até 2h.
- Não mande proposta.
- Proponha 2 horários concretos para call de 20 min: "Quarta às 10h ou quinta às 15h — qual funciona melhor?"
- O que acontece na call: `irbis-call-de-vendas`.
